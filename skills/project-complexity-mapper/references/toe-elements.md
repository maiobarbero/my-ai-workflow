# TOE Elements

Use this full checklist only for complete TOE assessment, or when fast scan needs
exact element names.

Score each element from 1 to 5:

1. None
2. Little
3. Some
4. Substantial
5. Very much

## Technical Elements

Technical complexity concerns project content: goals, scope, tasks, technology,
methods, deliverables, and technical risk.

| Element | What To Assess |
|---|---|
| High number of project goals | Number of strategic project goals, from single to many |
| Non-alignment of project goals | Degree of conflict between strategic goals, if more than one goal exists |
| Unclarity of project goals | How unclear goals are among team members |
| Uncertainties in scope | Uncertainty in agreed scope of work |
| Strict quality requirements | Extraordinary quality demands for deliverables |
| Project duration | Planned duration compared with a normal reference project |
| Size in CAPEX | Capital expenditure relative to the organization |
| Number of locations | Number of project sites or locations, including contractor locations |
| Newness of technology worldwide | Use of new or unproven technology for this application |
| Lack of experience with technology | Lack of experience among involved parties with the technology used |
| High number of tasks | Number of work packages, tasks, or subprojects |
| High variety of tasks | Degree of difference between task types |
| Dependencies between tasks | Number and strength of task dependencies |
| Uncertainty in methods | Uncertainty in technological methods or delivery methods |
| Involvement of different technical disciplines | Level of multidisciplinarity |
| Conflicting norms and standards | Conflicts between design standards, local norms, or country-specific rules |
| Technical risks | Number, probability, or impact of technical risks |

## Organizational Elements

Organizational complexity concerns the project team, internal organization,
resources, contracts, trust, methods, and organizational risk.

| Element | What To Assess |
|---|---|
| High project schedule drive | Pressure on schedule, from relaxed to should have finished already |
| Lack of resource and skills availability | Availability problems for required people, materials, or skills |
| Lack of experience with parties involved | Lack of prior work with partners, contractors, suppliers, or other parties |
| Lack of HSSE awareness | Lack of awareness of health, safety, security, and environment issues |
| Interfaces between different disciplines | Number of discipline interfaces that can create coordination problems |
| Number of financial sources | Number of funding sources, such as own investment, bank, subsidy, JV partner, or customer |
| Number of contracts | Number of contracts with customers, contractors, suppliers, or partners |
| Type of contract | Fit and consistency of contract types for the project |
| Number of different nationalities | Number of nationalities involved in the project |
| Number of different languages | Number of languages used in project communication |
| Presence of JV partner | Whether a joint venture partner is involved |
| Involvement of different time zones | Time zone spread and impact on coordination |
| Size of project team | Number of people in the project team |
| Incompatibility between PM methods or tools | Compatibility problems between project management methods or tools |
| Lack of trust in project team | Trust level among project team members |
| Lack of trust in contractor | Trust level in contractor or contractors |
| Organizational risks | Number, probability, or impact of organizational risks |

## External Elements

External complexity concerns context outside the project team: stakeholders,
politics, market, location, support, environment, and external risk.

| Element | What To Assess |
|---|---|
| Number of external stakeholders | Number of stakeholders outside the project team who influence or are influenced by the project |
| Variety of external stakeholders' perspectives | Degree of difference between stakeholder perspectives |
| Dependencies on external stakeholders | Number and criticality of dependencies on external stakeholders |
| Political influence | Degree to which political situation influences the project |
| Lack of company internal support | Lack of internal management support for the project |
| Required local content | Extent to which local parties must participate for permission or execution |
| Interference with existing site | Interference between current site use and project location |
| Remoteness of location | Reachability and availability of infrastructure or facilities |
| Lack of experience in the country | Lack of prior work experience in the country |
| Company internal strategic pressure | Internal strategic pressure from business or competing departments |
| Instability of project environment | Instability in exchange rates, raw material prices, economy, or context |
| Level of competition | Strength of competition in current market conditions |
| External risks | Number, probability, or impact of external risks |

## Common Fast-Scan Drivers

These drivers often surface early and are useful for fast scan:

- High schedule pressure
- Many external stakeholders
- Scope uncertainty
- Task dependencies
- Multiple disciplines involved
- Lack of resources or skills
- Interfaces between disciplines
- Political influence
- Trust gaps
- New or unfamiliar technology
- Contract complexity
- External approvals
