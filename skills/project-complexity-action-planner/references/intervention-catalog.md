# Intervention Catalog

Use this catalog to select actions from the diagnosed quadrant and TOE category.

## By Quadrant

### Simple / Internal

Use when detail and dynamic complexity are low.

Interventions:

- Name one accountable owner.
- Keep governance minimal.
- Use simple milestone check-ins.
- Avoid unnecessary workshops or heavy reporting.

Risk to manage:

- Hidden external or organizational complexity may be missed.

### Complicated / Systems Management

Use when detail complexity is high and dynamic complexity is low.

Control interventions:

- Work Breakdown Structure.
- Interface register.
- Dependency map.
- Requirements baseline.
- Decision gate.
- Quality checklist.
- Risk register.
- Contract responsibility matrix.
- Milestone plan.
- Change control threshold.

Success signals:

- Interfaces have owners.
- Dependencies are visible.
- Acceptance criteria are explicit.
- Decisions are recorded.
- Variance is reviewed before it compounds.

### Complex / Interactive Management

Use when detail complexity is low or medium and dynamic complexity is high.

Connecting interventions:

- Problem-framing workshop.
- Scenario session.
- Tradeoff session.
- Short feedback loop.
- Stakeholder alignment forum.
- Transparent decision log.
- Escalation path for conflicts.
- Assumption review.

Success signals:

- Actors share enough language to decide.
- Conflicts surface early.
- Assumptions are visible.
- Expectations are not inflated.
- Scope changes are deliberate.

### Complex + Complicated / Dynamic Management

Use when detail and dynamic complexity are both high.

Mixed interventions:

- Interface register plus joint interface workshops.
- Decision gates plus assumption reviews.
- Requirements baseline plus controlled discovery window.
- Risk register plus joint risk workshop.
- Contract matrix plus escalation forum.
- Technical prototype plus stakeholder playback.
- Phase gate plus complexity reassessment.

Success signals:

- Control artifacts reflect real stakeholder constraints.
- Interaction produces decisions, not endless discussion.
- Technical dependencies and actor dependencies are managed together.
- Leadership balances predict-and-control with prepare-and-commit.

## By TOE Category

### Technical

Useful interventions:

- Architecture decision record.
- Prototype risky technology.
- Interface register.
- Dependency map.
- Requirements baseline.
- Acceptance criteria.
- Technical risk review.
- Cross-discipline design review.

### Organizational

Useful interventions:

- RACI or responsibility matrix.
- Resource and skills gap review.
- Decision rights clarification.
- Contract responsibility matrix.
- PM method/tool alignment.
- Team trust repair session only when trust is diagnosed as issue.
- Schedule pressure review with scope tradeoffs.

### External

Useful interventions:

- Approval map.
- Political/context risk review.
- Public trust or adoption strategy.
- External dependency tracker.
- Scenario planning.
- Sponsor alignment.
- Regulatory/compliance gate.
- Value reframing session.

## Value Opportunities

When diagnosis shows value opportunity, preserve it in the plan.

Examples:

- Turn stakeholder support into adoption.
- Turn compliance into trust.
- Turn interfaces into reusable assets.
- Turn funding constraint into scope reframing.
- Turn external pressure into clearer decision mandate.

Do not force value creation if diagnosis does not support it.
